## 2) ELMATADORZ AGENTIC FLOW (10 STAGES)
### Stage 0 — Intent Lock
Lock: Goal / Deliverable / Constraints / Forbidden / Tone.  
**Artifact:** Intent Contract (IC)

### Stage 1 — System Boundary
Boundary, stakeholders, incentives, hidden costs, feedback loops, delays.  
**Artifact:** System Map (SM)

### Stage 2 — First Principle Decomposition
Axioms / Invariants / Variables / Levers + **Falsifiers**.  
**Artifact:** First Principle Frame (FPF)

### Stage 3 — Observe (Evidence Inventory)
Known / Unknown / Data risks / Signal-vs-noise criteria.  
**Artifact:** Evidence Sheet (ES)

### Stage 4 — Orient (Minimum Viable World Model)
Loops, delays, bottlenecks, constraints, causal structure.  
**Artifact:** World Model (WM)

### Stage 5 — Decide (3-tier Options)
Option A Safe / Option B Balanced / Option C Aggressive  
Each must include: upside, downside, reversibility, triggers, kill-switch.  
**Artifact:** Decision Set (DS)

### Stage 6 — Act (Staged Execution)
Staged actions with rollback, metrics, thresholds, kill-switch.  
**Artifact:** Action Plan (AP)

### Stage 7 — Audit Gate (Structural Self-Verification)
No output is allowed without passing 5 gates:

1) **Truth Gate** — facts vs assumptions vs claims  
2) **Logic Gate** — no leaps, no missing premises  
3) **Risk Gate** — bounded downside + kill-switch + rollback  
4) **Bias Gate** — confirmation / sunk cost / narrative fallacy / authority bias  
5) **Clarity Gate** — one-page explainable + executable checklist  

**Artifact:** Audit Report (AR) + PASS/FAIL

### Stage 8 — Signed Memory (Continuity)
Append-only record: claims, decisions, assumptions, falsifiers, outcomes.  
**Artifact:** Signed Log (SL)

### Stage 9 — Learning Loop (Upgrade)
If falsified → update model  
If weak performance → adjust levers/metrics  
**Artifact:** Model Upgrade (MU)

# PART D — SELF-AUDIT DSL (SIMPLE RULE LANGUAGE)
This is a lightweight DSL for structural verification.

## Grammar (human readable)
RULE <name>:
  REQUIRE <field>
  FORBID <pattern>
  FAIL_IF <condition>
  FIX <instruction>

## Default Ruleset
RULE MustHaveFalsifiers:
  REQUIRE FPF.falsifiers
  FAIL_IF len(FPF.falsifiers) == 0
  FIX "Add at least 3 falsifiers + 3 minimal tests."

RULE MustBoundDownside:
  REQUIRE DS.GlobalDownsideBound
  FAIL_IF DS.GlobalDownsideBound is empty
  FIX "Define downside bound and kill-switch triggers."

RULE MustSeparateFactsAssumptions:
  REQUIRE ES.Facts
  REQUIRE ES.Assumptions
  FAIL_IF len(ES.Facts)==0
  FIX "Write facts separately from assumptions."

RULE MustPassAuditToAct:
  REQUIRE AR.Verdict
  FAIL_IF AR.Verdict == "FAIL"
  FIX "Resolve failed gates before action."

RULE MustBeOnePageExplainable:
  REQUIRE AR.ClarityGate
  FAIL_IF AR.ClarityGate is empty
  FIX "Compress explanation into one page + checklist."
